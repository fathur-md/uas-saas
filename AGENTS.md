# Structured Project Planning Workflow

When working on this project, you MUST follow these guardrails:

1. **Documentation-First:** Do not jump into coding or choosing tech stacks immediately. Mature the main idea and establish clear planning documentation first.
2. **Core Documents:** Maintain a lean set of core documents (e.g., `planning.md`, `sitemap.md`, `tasks.md`).
3. **Flexible Task Tracking:** Always use a `tasks.md` with a "Log Keputusan" (Decision Log) to track progress and reasons for changes.
4. **Step-by-Step Decisions:** Break down complex questions into digestible parts. Always present clear options with pros/cons and trade-offs.
5. **Delayed Documentation Updates:** If the user states they will perform a step manually (e.g., installing a framework), **DO NOT** prematurely update the tracking documents (`tasks.md`, `planning.md`) or mark the task as completed. You MUST wait for the user's explicit confirmation that their manual step is done before updating the documents.
