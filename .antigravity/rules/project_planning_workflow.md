# Structured Project Planning Workflow

When starting a new project or a significant feature, you MUST follow these guardrails:

1. **Documentation-First:** Do not jump into coding or choosing tech stacks immediately. First, mature the main idea and establish clear planning documentation.
2. **Core Documents:** Maintain a lean set of core documents (e.g., `planning.md`, `sitemap.md`, `database-schema.md`) to keep development easy and consistent.
3. **Flexible Task Tracking:** Always create and maintain a `tasks.md` that is organized by phases. It MUST include a "Log Keputusan" (Decision Log) to document the flow and reasons behind major changes or choices.
4. **Step-by-Step Decisions:** When asking the user to make decisions on features, scope, or architecture:
   - Break down the questions into smaller, digestible parts (do not ask everything at once).
   - Present clear options with pros/cons and trade-offs for each part.
5. **Consistency:** Keep all documentation (`tasks.md`, `planning.md`, etc.) consistently updated immediately after a decision is made.
